=========================================================
Purpose of this file
=========================================================
The porpose of this file is to describe the layout and 
functionality of Buy and Sell, a web/native app based 
system that can be used to buy and sell second hand stuff.

The system also have a page for services, where service
providers can advertise their services for a fee.

System = the code and database combination that operates
together as Buy and Sell

The System does not accept payments. The system only
facilitates the buying and selling, providing a platform
for this purpose.

The system generates money through strategically placed
adds, which does not distract from the user experience.

Although i am providing you with the framework of the app
and its functionality, you are welcome to make design changes
to make it work and look better!

So we have these components:

Database
API

Web App

Native React App

Please build the Database and Server together so its easy to test locally

Then build the Web app in react seperate, which should be packaged to work on the desktop

Then build the Native React App for the smartphone

Please ask me any questions for clarification if you have any during the design
and build phase.


=========================================================
Database
=========================================================
The database is the core of the system containing all the 
information the system use to operate. Lets use PostGRESQL 
for the database.

Database Schema

#this table holds the client information and also the username
and password which is used for login. This is the required information
the client needs to provide during sign up. After signup, the client
get a verification email. Upon correct verification, the client_verfied
is set to true.

Must start with a admin account already setup

TABLE client (
    client_id,
    username,
    password,                 #needs to verify 6 chars and secure, needs to be entered twice to check if no mistake has been made
    email,                    #needs validation
    whatsapp,                 #input field needs to allow country to be selected
    active :boolean,          #start value = true
    join_date,
    last_login :date,
    location_id,
    client_role_id            #on form client can select between {Basic,Silver,Diamond}
    client_verified: boolean, #start value = false
)

#this is the clients subscription level. Differend subscriptions place the clients listed products higher on the
product listing. lets make Basic = free, Silver = P50, Diamond = P100. Currency is as per location

TABLE client_role (
    client_role_id,
    client_role,
    sub_price
)

#this table holds all the currencies, we can start with Rand and Pula
Table currency (
    currency_id,
    currency_full_name,
    currency_short
)

#this table holds a list of the towns where clients are located and can sell from. the town radius is going to 
test if someone is from within the town limits they are selling from
TABLE location (
    location_id,
    lacation_name,    #name of town/city
    currency_id,      
    location_lat,     #latitude of location
    location_long,    #longitude of location
    location_radius   #circle from centre of town to town limits and surrounds  
)

#this table holds the information of the product the client is selling. On the Create Products page
the client would be able to take a picture, write a small description of what it is, and state a product_price
and the category in which it should be listed.

TABLE product (
    product_id,
    product_description,
    product_image :BLOB,
    product_price,
    category_id
)

#this table holds the product listing, where all the products are listed. The products posted by clients with
a subscription, will be listed at the top, depending on their tier, when displayed in the app. The product can only 
be avail on the listing for 3 days after which it is moved to dormant status and the owner will see the notification
on the app and they will have the option to move it to the active state again. Once the product has been dormant 
for 7 days, it is reomoved from the database. The product can only be re-instated 2 times, thereafter it would be
deleted from the database.

TABLE product_listing (
    listing_id,
    product_id,
    client_id,
    buyer,                      #this would be the client_id of whoever buy the item
    status,                     #status will be {avail,sold,dormant} start value = avail
    listing_date                #date the product were added to the listing
    listing_reinstate_counnt :int
    products_position_on_list  #depending on the clients role, how close to the top their product would be shown
)

#this table states the category of the product. You can please help me come up with 12 categories, for example:
{Appliances, Vehicles, Boats, Kitchen Ware, Building Hardware} and so on.
TABLE product_category (
    category_id,
    product_category

)

#this is the service table. On the service page of the app the client will have the opeprtunity to create a service.
this is a directory for businesses to show make the visible. With a subscription, the clients service can be listed 
towards the top of the directory. Higher tiers, higher up. The client_role determine this.
TABLE service (
    service_id,
    client_id,
    service_logo :BLOB,   #should be able to take an image, you descide formats, size and crop for a customers logo
    service_name,
    service_description,
)

#this table holds the service listings which would be displayed on the app.
TABLE service_listing (
    service_listing_id,
    service_position_on_list,   #client_role determines where on the service list it would be listed
    service_id,
    service_status : boolean,
    service_listing_date,
)

#if a client has a Diamond role, their service would be advertised on the space set out for this at the bottom of
the deifferent screens. this advertisement, would flip through the advert_list in sequence, giving each at least 
10 seconds of screen time.
TABLE advert_list (
    advert_list_id,
    service_id,
    service_clicks: int,
)
